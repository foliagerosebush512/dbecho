## 🔍 How dbecho Works

dbecho connects to your PostgreSQL databases using tools that speak the MCP (Model Context Protocol) standard. It lets you:

- Explore database schema easily  
- Run analytics without changing your data  
- Compare data from different databases side by side  

The app runs locally and only reads data. It does not change or write to your databases.

---

## 🛠️ Connecting dbecho to Your Database

To use dbecho, you will need:

- The address of your PostgreSQL server (for example, `localhost` or an IP address)  
- Your database name  
- User name and password for your database account  

Steps to connect:

1. Open dbecho.

2. Click on “Add New Connection.”

3. Enter your PostgreSQL server address.

4. Enter your database name.

5. Enter your username and password.

6. Click “Connect.”

dbecho will check your connection and open the database for exploration.

## 📊 Using dbecho Features

Once connected, you can:

- View tables and columns in your database schema  
- Browse data inside tables  
- Run simple analytics like counts and summaries  
- Compare tables across different databases side by side  

The interface is clean and designed for ease of use. If you want to use another MCP client like Claude Code or Cursor, dbecho supports that too.

## 🔧 Troubleshooting Tips

- If dbecho does not connect, check your database server is running and that your login details are correct.  
- Firewalls can block connections. Make sure your PC can reach the database server on port 5432 (default for PostgreSQL).  
- If the installer fails, try running it as Administrator (right-click, select “Run as administrator”).  
- If you get errors inside dbecho, restart the app and try reconnecting.
